- **Intermittent engine-init segfault (`Creating Engine` → `ready` → `gjs exited with code null`)** — the remaining fragility for *reliable* multi-instance / automated runs (op-sync itself is correct). Hits either peer on `open-scene` engine init, roughly 1 in 3 launches; succeeds on retry. Running under `gnome-session-inhibit --inhibit idle` (screen awake / DPMS off) markedly reduces it — a dark-screen / no-frames GLArea is a big factor — but doesn't eliminate it, so there's also a GL/libepoxy init race. Repro: launch two instances, drive both through `open-scene`, watch for the `code null` exit. Likely fixed by the same **offscreen-GL** work that would enable a headless peer; related to the "Fallback disabled" libepoxy note in `packages/gjs/src/widgets/engine/engine.ts`. *owner: engine / gjs / gjsify, why: makes pair-edit / automated collab flaky*
